# 三、表单完整校验规则
全局通用校验
标题、地点、描述不能为空
二手交易专属校验
商品分类必填；价格数值必须大于 0；成色选项不能为空
失物招领专属校验
物品名称、事件发生时间不能为空
拼单搭子专属校验
拼单类型必填；目标招募人数≥2；截止时间不能为空
跑腿委托专属校验
任务类型、取件地、送达地、截止时间不能为空；酬劳不能为负数

# 七、开发问题与解决方案
问题 1：切换发布类型，旧表单数据残留
原因：切换分类时表单对象未清空，旧字段值保留
解决：使用 watch 监听 publishType，每次切换自动执行 resetForm 清空表单、清除校验报错
问题 2：点击提交按钮页面整页刷新
原因：form 标签原生提交事件触发页面重载
解决：表单绑定@submit.prevent="handleSubmit"阻止浏览器默认提交行为
# 八、今日开发反思
统一发布表单是平台核心用户录入入口，本次开发完整走通「前端录入 - 本地校验 - 接口提交 - 页面跳转」标准 CRUD 流程。
动态表单依靠 Vue 条件渲染，一套组件兼容四类业务场景，大幅减少重复代码；前置表单校验能够拦截非法数据，减轻后端存储压力，避免脏数据。
同时意识到 AI 生成代码仅能完成基础骨架，在持久化存储、交互体验、全局状态对接、代码复用性上仍需人工优化调整。该表单开发模式具备通用性，后续新增发布类型可直接复用当前结构快速拓展。
